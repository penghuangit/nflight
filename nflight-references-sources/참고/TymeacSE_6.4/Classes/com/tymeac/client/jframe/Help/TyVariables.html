Tymeac Server RMI Variables

This Class contains the String names by which Tymeac Server and Clients communicate.

When you change any of the instance fields, you must recompile this Class into the <home>\Common directory. Both the Tymeac Server and Tymeac GUI and non-GUI Clients use this class. Your clients may also use this class when communicating with the Tymeac Server. Naturally, both the Server and Clients must use the same values for communication.

Basic RMI Panel:

private static String tymeac_url_name -- This is the URL for RMI calls.

By default the name is for local calls "rmi:///".

Note the three slashes (///).  This is not a typo. That third slash is a trailing slash. In order to set this to a remote name, you put that remote name between the second and third slashes.  Such as:
rmi://your-company.org/  or  rmi://192.168.42.1/

private static String tymeac_name --

By default the name is mixed case "com.tymeac.demo.TyDemoActivation".

For non-activatable remote objects:

The RMI Naming.rebind() name used by Tymeac Server at start up and by all Tymeac Clients in RMI Naming.lookup(). You may change this to your installation requirements. This class ships with the name set as above. For non-activatable remote objects there is no requirement for the String value. The Server binds to the RMI Registry using this String value and Clients lookup using this same String value.

For activatable remote objects:

The name of the Activatable Object Class. This is the Class that extends TymeacActivationImpl. This name is used by the Java Activation System, (rmid or Pheonix), to activate the Tymeac Server and by the Tymeac Clients to request a Tymeac Server function. This class ships with the name set as above. This is for the demonstration system Activatable Object Class.

IIOP Panel:

This field is for the JNDI Registry, (CosNaming server), Communication, (RMI-IIOP).

private static String tymeac_iiop_name --

By default null.

The JDNI Naming.rebind() name used by Tymeac Server at start up, (InitialContext().rebind() ), and by all Tymeac Clients in CosNaming.lookup(), (javax.naming.lookup() ).  You may change this to your installation requirements.

Alternate Panel:

This field is for non standard Communication. 

private static String tymeac_alternate_class_name -- The name of the non standard communication protocol class.

By default null.

Jini, release 1.x is such a protocol. Since there is no way to package such a method, the execution is up to the user. You must write a class that implements your way of communication. Place the name in the String tymeac_alternate_class_name.
