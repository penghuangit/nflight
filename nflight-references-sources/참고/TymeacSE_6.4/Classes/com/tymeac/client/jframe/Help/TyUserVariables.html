Tymeac User Variables Class Maintenance

This Class produces a simple java file with the individual elements, described below, as a text line followed by: System.getProperties().getProperty("line.separator").

DO NOT EDIT THIS FILE  The parsing algorithm for this file is very simple and cannot deal with extra spaces and comments.

General Pane:

Elements

Sys.exit():      This Checkbox is Not Checked for no shut down thread, Is Checked for yes. The RMI server is keep alive by a thread that never ends.  In order to end Tymeac Server after a normal shut down, Tymeac starts a thread at start-up.  This thread sleeps until awakened at Tymeac shut down.  It then calls System.exit(0) which ends the Java Virtual Machine. Not Checked for this option does not start this thread. The file line is either the String "yes" or "no".

Monitor Interval:    The interval, in seconds, that the Tymeac Monitor sleeps.  Another way of saying this is the delay between running of the Monitor.  When zero, Tymeac does not start the Monitor.  Tymeac also uses this interval for timing of how long requests are in the system.  When over this (interval (times a) factor) a request may be marked as possibly stalled.   A good place to start is 60 seconds.

Inactivation Minutes:  This is the time, in minutes, to inactivate the Tymeac Server when no activity takes place.  This is only valid for the Remote Object Activation System and when specifying a Monitor Interval, above.

It is the Monitor that checks for time of activity.  Therefore, there is a close correlation between the Monitor interval and the Inactivation Minutes.  If one specifies an Inactivation time of one (1) minute and a Monitor interval of ten minutes (600 seconds), then the Inactivation time in effect becomes ten minutes.

Notification Function:    This is the Tymeac Function the Monitor executes when it finds a potential problem.  The Monitor sends a message to this Function. This is a customer responsibility.  Tymeac supplies a sample for your perusal.

Buttons

BEGIN:    This button brings up the standard FileDialog Menu.  The default FileName is TymeacUserVariables.java.  When no file already exists, bypass this button. This is to import an existing file.

FINISH:    This button brings up the standard FileDialog Menu.  The default FileName is TymeacUserVariables.java.. YOU CANNOT ALTER THIS NAME  This is to update the file, or add a new file.

Security Pane:

Elements

Login Context:   This is the String for a login context, or null.

Exits Pane:

Elements

Startup Classes List:   These are the classes Tymeac instantiates at startup.

Startup Classes TextField:  This is the TextField for adding a String to the above start up list.

Startup Functions List:   These are the Functions Tymeac starts at startup.

Startup Functions TextField:  This is the TextField for adding a String to the above start up functions list.

Stage 1 shut down Classes List:    These are the classes Tymeac instantiates at stage 1 of shut down.

Stage 1 shutdown TextField:  This is the TextField for adding a String to the above stage 1 shut down list.

Stage 2 shut down Classes List:    These are the classes Tymeac instantiates at stage 2 of shut down.

Stage 2 shutdown TextField:  This is the TextField for adding a String to the above stage 2 shut down list.

Buttons

ADD:  This button adds the TextField to the associated List.

REMOVE:  This button removes the selected String from the associated List.

ALTERNATES

Elements

Files

Statistics Directory:  This is the TextField for using a file directory.  When using a file to store the statistics, this is optional.

Statistics File:  This is the TextField for using a file name.   When using a file to store the statistics, this is required.

Log Directory:  This is the TextField for using a file directory.   When using a file to store the log, this is optional.

Log File:  This is the TextField for using a file name.   When using a file to store the log, this is required.

 

Classes

Statistics Classes:  This is the TextField for using an alternative class for the statistics repository.

Log Classes:  This is the TextField for using an alternative class for the log repository.
