Tymeac Installation

 

Tymeac does not require any alteration to your basic environment other than setting a CLASSPATH, when necessary.

Create a Tymeac Home directory <home>:
    x:\Tymeacxx.x.x or
    usr/local/tymeacx.x.x
and unzip the package into this directory.

When you unzip the file, Tymeac configures into a <home> directory, with sub directories.

The starting place is in the Tymeac documentation section, index.html, below.

1.  <home>/Classes/com/tymeac  This directory contains the Tymeac Classes. The sub directories are:

/Base  Classes that are necessary to run the Server.

/Client  Classes that are necessary to run the non-GUI clients.

/JFrame  Classes that are necessary to run the Tymeac GUI Clients, (using Swing.)

/Help Help files for the GUI Clients.

/Images image files for the GUI Client help files.

/Common   User Classes that are common to both the Server and the Client

/ServerUser  Classes for the Server Processing Application Classes.

/Demo  Classes that are for the Tymeac Client, for demonstration only. There are sub-directories therein for Jframe, Jini and Jini2.

/Jini  Classes that are for Jini, version 2.x. There are sub-directories therein for base and client.

/Security  One "full permission" policy file.

2.  <home>/Source  src.zip  The complete source files for the product.

3.  <home>/Doc  Tymeac documentation.

The documentation, Doc/index.html is the starting place for the set-up and Installation Verification Procedure.

Go to the HTML format starting place,  Doc/index.html.

Follow the link to the Technical Reference Manual,
then Getting Started.

The link, Product Overview gives a basic overview of the Tymeac Product.

The link, Setting up the Environment is the starting place to complete the installation of the Product.  This explains the directory structure mentioned above along with using a CLASSPATH or -cp.

4.   <home>/Scripts  Start up scripts for the Tymeac Server.
