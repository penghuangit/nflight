Tymeac User Functions

The purpose of this Class is a replacement for the Function and List Tables in a data base management system and the maintenance of that table by the frame Class, TyFuncMaint.

Since this is a user maintained Class, we provide a simple, source code Class, (FunctionChecker), with which you may check the validity of this Class before starting the TymeacServer.

Since maintaining this class by hand can result in errors, we provide a GUI with which you may maintain this class. This is a binary "use the GUI or do it by hand". If you use the GUI, then you should not try to edit the resulting java file. The GUI is very simple -- it reads a java file without comments or formatting.

In either case, once you create the TymeacUserFunctions.java file, you must compile it into the com/tymeac/serveruser directory.

The GUI class is com.tymeac.jframe.TyUserFunctions.


(Click for full image)

It require access to the java file, TymeacUserFunctions.java. You may not substitute another file name in the dialog box.

The editing for the fields is the same as the DBMS Functions GUI, TyFuncMaint except that there is no verification of output agent and normal Queues.

The rest of this document explains the java file format.

 

Instance Fields

Int total_functions - The total number of Tymeac Functions in the system.

com.tymeac.base.TymeacIndividualFunction[] tif - This array contains each Function.

 

Constructor

This is where you set the values for the above instance variables.

You may either use the no-argument constructor for the TymeacIndividualFunction Class and set all the fields with mutators, or use the argument constructor to set the field values.  The source code we supply with the product demonstates both methods.

 

Instance Methods

These are accessors for all the above instance fields.
