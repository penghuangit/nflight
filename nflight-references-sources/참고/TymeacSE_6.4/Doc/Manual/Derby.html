Using Derby

Derby is an optional part of the JDK download. You may either download it with the JDK or from the apache.org website and install it on the machine on which you intend to use it. This document assumes you have installed Derby and set up DERBY_HOME etc.

Tymeac setup

Easy: (If you use NetBeans...)

You may use NetBeans 5.0+ to build the database and tables. Go to:
     tools > options > advancedOptions and under Advanced Options:
          IDE Configuration / Server and External tool settings / Derby database

Click on Derby database. Fill in the directory where you installed Derby and the place where you want the database located.

Go back to tools, expand and choose Derby Database > Create ...The name of the database is upper-case TYMEAC.

Once built, go back to tools, expand and choose Derby Database > Start...

Now click on the runtime icon on the left of the screen or use menu bar item Window and choose runtime. This brings up all the running elements. Expand databases. Right click the Derby URL and choose connect...

Fill in the user name and password, if you chose to use such when you created the database, and click ok.

Once connected, right click TABLES, choose create table and follow the directions for creating a table and elements therein. The Tymeac table names and elements are found starting from here.

Once all the tables are built, it's time to fill in the data for the demonstration system, see below.

Standard:

Follow the instructions for creating a database in the Derby Getting Started Guide. The name of the database is upper-case TYMEAC. You may or may not use a username and password.

The Tymeac table names and elements are found starting from here. We also supply sql statements for creating the tables. These are in the /scripts/sql directory. Once the database exists, use ij to connect and then run the sql scripts, such as: ij> run 'QUEUETABLE.SQL';

Once all the tables are built, it's time to fill in the data for the demonstration system, see below.

Data:

In order for Tymeac to find the names of the various tables and DBMS information, Tymeac uses a configuration file or preference data. Before you can use the Queue and Function GUI classes you must create the file or preference data. Use the GUI to do this.

There are two tables necessary for the demonstration system. The Queue Table, where the Queue data lives and the Function Table where the Function data lives.

You must do the Queues first since adding Functions requires the Queues to exist. For Queues see here. For Functions, see here.

Scripts:

All scripts are in the script directory.

You must set the SET_DERBY value in the supplied script.

That's all there is to it.
