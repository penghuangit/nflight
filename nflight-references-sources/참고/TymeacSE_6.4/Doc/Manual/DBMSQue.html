DBMS Tymeac Queue Table

This table contains all the Queues in the Tymeac system. All column names are UPPER CASE, must be in the order presented here, and must be spelled as presented here. Variable Character columns convert into Java Strings, therefore, lengths are customer determined. 

See the Tymeac Queue Maintenance Class (TyQueMaint) for a description of values for each column.

There is a pre-built definition in the /scripts/sql directory for this table to make your life a little easier.

Keys:  

Single key -- QUEUE_NAME

Columns:

QUEUE_NAME    Var Char.    The name of a Queue.

PA_CLASS    Var Char.    The name of the Processing Application Class.

NBR_WAITLISTS   Integer.    The total number of Wait Lists for this Queue.

NBR_IN_WAITLISTS   Integer.    The number of physical entries in each Wait List.

WAIT_TIME   Integer.    The maximum time, in seconds, the Queue Thread waits for a Client posting before becoming inactive.

THREAD_KILL_TIME  Integer.   The maximum time, in seconds, the Queue Thread waits as an inactive thread, (see above, WAIT_TIME), before a thread-destroy, (return in the run() method.)

NBR_THREADS   Integer.    The maximum number of Threads for this Queue.

QUE_TYPE   Integer.    The type of Queue, 0-normal, 1-Output Agent.

THREAD_START_DELAY  Integer.  Start all threads at Tymeac Start Up, 0-no, 1-yes.

OVERALL_PERCENT    Double.    The Thread Threshold overall percent.

INDIVIDUAL_PERCENT   Double.    The Thread Threshold Individual percent.

WEIGHTED_FACTOR   Double.    The Thread Threshold weighted factor.

WEIGHTED_AVERAGE   Double.    The Thread Threshold weighted average.

PA_TIMEOUT  Integer.    The time, in seconds, a Processing Application Class may execute before Tymeac believes it may be stalled.   Zero -- use the default, greater than zero, use this number.

TH_NBR_IN_WAITLISTS  Integer.    The number of logical entries in each Wait List. Since release 6.0
