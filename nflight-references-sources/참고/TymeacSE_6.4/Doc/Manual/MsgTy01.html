Messages: Start Up Config Extension Ty01

This section details the Start Up messages when using a Configuration Provider.

(P) Printed to System.out.println(), (this is optional).
(L) Sent to the Tymeac Log.
(N) Sent to the Tymeac Notification Function.

003 S (P)

Tymeac Server No command line arguments

Use of command line arguments is mandatory. See the start up document for command line usage.

005 S (P)

Tymeac Server No configuration (-config) argument

Use of a configuration provider is mandatory. See the start up document for command line usage.

010 S (P)

Tymeac Server No configuration parms after -config

Use of a configuration provider is mandatory. See the start up document for command line usage.

015 S (P)

Tymeac Server ConfigurationProvider error=xxxx

A configuration provider error. xxxx is the reason. xxxx is the reason.

020 S (P)

Tymeac Server Configuration getEntry for Directory error=xxxx

A configuration getEntry() method returned this error while attempting to access the directory entry in the configuration file. xxxx is the error. See the start up document for command line usage.

025 S (P)

Tymeac Server Configuration getEntry for File error=xxxx

A configuration getEntry() method returned this error while attempting to access the file entry in the configuration file.xxxx is the error. See the start up document for command line usage.

030 S (P)

Tymeac Server Configuration getEntry for Stand Alone error=xxxx

A configuration getEntry() method returned this error while attempting to access the stand alone entry in the configuration file.xxxx is the error. See the start up document for command line usage.

040 S (P)

Tymeac Server Configuration getEntry for Verbose error=xxxx

A configuration getEntry() method returned this error while attempting to access the verbose entry in the configuration file.xxxx is the error. See the start up document for command line usage.

041 S (P)

Tymeac Server Configuration getEntry for Embedded Data Base Shut Down error=xxxx

A configuration getEntry() method returned this error while attempting to access the edbsd entry in the configuration file.xxxx is the error. See the start up document for command line usage.

045 S (P)

Tymeac Server No ConfigurationProvider

A configuration getEntry() method returned this error while attempting to access the xxxx entry in the configuration file. xxxx is the error. See the start up document for command line usage.

050 S (P) (L)

Tymeac Server Configuration getEntry error=xxxx

A configuration getEntry() method returned this error while attempting access in the configuration file. xxxx is the error.

055 S (P) (L)

Tymeac Server Discovery manager failure=

Jini --An access to a discover manager produced the error.

060 S (P) (L)

Tymeac Server Join Manager failure=

Jini --An access to a join manager produced the error.

065 S (P) (L)

Tymeac Server No application entry, xxxx

Jini -- Activation create, field xxxx is missing. In order to create a backend object for the Activation System to activate, certain entries must be in the Configuration File. A field is missing.

070 S (P) (L)

Tymeac Server Activation creation, unbalanced properties

Jini -- Activation create, the properties field  (String[]) in the configuration file needs an even number of entries. E.G. "java.security.policy", "where located".

Make sure all the String properties are matched.

075 S (P) (L)

Tymeac Server Activation creation, Error creating Marshaled Object: xxxx

Jini -- Activation create, the options field  (String[]) in the configuration file contains Strings used to create a Marshaled Object for the activated program (command line replacements). xxxx is the error from that create.

080 I (P) (L)

Tymeac Server Created Tymeac Activatable environment successfully

Jini -- The backend activation system was created and registered with the RMI Registry successfully.

081 S (P) (L)

Tymeac Server Activatable environment creation failure

Jini -- Activation create failed. There should be prior messages to indicate why.

085 S (P)

Tymeac Server Activatable RMI Registry failure= xxxx

Jini -- Activation create phase 1 requires using the RMI Registry as a temporary repository for the backend remote object. The phase 2 picks up the remote object. A Naming.rebind failed.

090 S (P)

Tymeac Server Activatable environment create Exception= xxxx

Jini -- Activation create phase 1. Caught exception. xxxx is the exception.

095 I (P)

Tymeac Server Activatable Jini Service shut down

Jini -- Activation Service ended normally.

100 I (P)

Tymeac Server Activatable Jini Service created successfully, waiting for a shut down.

Jini -- Activation Service is up and registered and will remain until shut down.

105 S (P)

Tymeac Server Activatable RMI Naming failure=xxxx.

Jini -- Activation .Activation create phase 1 requires using the RMI Registry as a temporary repository for the backend remote object. The phase 2 (now) picks up the remote object. A Naming.lookup failed.

110 I (P)

Tymeac Server Activatable Tymeac Join Manager terminated

Jini -- Activation. The Tymeac service terminated

115 I (P)

Tymeac Server Activatable Internal Join Manager terminated

Jini -- Activation. The internal service terminated. This is an internal Jini service that accepts shut down requests from the Tymeac Server or another Jini program for destroying the service. See the start up document for a detailed discussion
