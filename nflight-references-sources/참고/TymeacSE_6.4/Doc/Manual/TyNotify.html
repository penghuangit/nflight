Tymeac Notification Function

Tymeac Notification is based on the principle that things can go wrong and those responsible for the smooth operation of a computer facility need an early warning system.

The notification system is optional. For the Demonstration System, Tymeac provides:

a sample notification Function, TyNotify,
a sample notification Queue, Notify, and
a sample Notification Class, com.tymeac.serveruser.DemoNotify.

There is a second sample that is the same as above with a 2 appended to the end (TyNotify2, etc.) You may use this to swap between notification functions dynamically.

The Tymeac Monitor  examines the internal structure looking for actual or potential problems. An actual problem is that all Queue Threads are 'disabled'. A potential problem is that a thread's processing or wait time is excessive. The Monitor sends a String message describing the problem to this Tymeac Function as an Asynchronous Request. Further processing is your option.

During Tymeac start up, the Server verifies that the Notification Function is valid. The Server sends a null message to this Tymeac Function as a Synchronous Request and examines the return String for the Tymeac return code.

Therefore, when writing your own Notification Class, handle the null or zero length message and just return. The sample Notification Class, DemoNotify, does this.

The name of the Notification Function is set in the Tymeac Configuration File, TyCfg or UserVariables Class. You can use our sample class as a starting point for developing your own early warning system or as an interface to an existing system.

Notification is dynamically alterable.
