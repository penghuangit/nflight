Tymeac Server RMI Variables

This Class contains the String names by which Tymeac Server and Clients communicate.

We supply the source code for this Class in the "Source Directory" the installation creates.

  If you change any of the instance fields, then you must recompile this Class into the <home>\Common directory. Both the Tymeac Server and Tymeac GUI and non-GUI Clients use this class. Your clients may also use this class when communicating with the Tymeac Server (see TySvrComm). Naturally, both the Server and Clients must use the same values for communication.

However, when you develop your own systems, you may have several of these classes in use as well as alternatives.

Tymeac Server uses the accessor methods, below, therefore, these may not change. We do not recommend adding any methods.

Since maintaining this class by hand can result in errors, we provide a GUI with which you may maintain this class. This is a binary "use the GUI or do it by hand". If you use the GUI, then you should not try to edit the resulting java file. The GUI is very simple -- it reads a java file without comments or formatting.

In either case, once you create the TyVariables.java file, you must compile it into the com/tymeac/common directory.

The GUI class is com.tymeac.jframe.TyVariables.

Fields:

These fields are indicators for which method to use:

  • the RMI Registry
  • the JNDI Registry (CosNaming server for RMI-IIOP or ORBd for the Portable Object Adapter)
  • a user defined alternate method

private static boolean tymeac_use_rmi

By default:  true

Use the RMI Registry to locate the remote reference to the Tymeac Server.

private static boolean tymeac_use_iiop

By default:  false

Use the JNDI Registry (CosNaming server/ORBd) to locate the remote reference to the Tymeac Server.

private static boolean tymeac_use_alternate

By default:  false

Use an alternate method to locate the remote reference to the Tymeac Server.

Basic RMI:

These two fields are for RMI Registry Communication.

private static String tymeac_name --

By default the name is mixed case "com.tymeac.demo.TyDemoActivation".

For non-activatable remote objects:

The RMI Naming.rebind() name used by Tymeac Server at start up and by all Tymeac Clients in RMI Naming.lookup(). You may change this to your installation requirements. This class ships with the name set as above. For non-activatable remote objects there is no requirement for the String value. The Server binds to the RMI Registry using this String value and Clients lookup using this same String value.

For activatable remote objects:

The name of the Activatable Object Class. This is the Class that extends TymeacActivationImpl. This name is used by the Java Activation System (rmid) to activate the Tymeac Server and by the Tymeac Clients to request a Tymeac Server function. See the activation system set up instructions. This class ships with the name set as above. This is for the demonstration system Activatable Object Class.

private static String tymeac_url_name -- This is the URL for RMI calls.

By default the name is for local calls "rmi:///".

Note the three slashes (///). This is not a typo. That third slash is a trailing slash. In order to set this to a remote name, you put that remote name between the second and third slashes.  Such as:
rmi://your-company.org/  or  rmi://192.168.42.1/

IIOP:

This field is for the JNDI Registry (CosNaming server/ORBd) Communication (RMI-IIOP).

private static String tymeac_iiop_name --

By default null.

The JDNI Naming.rebind() name used by Tymeac Server at start up (InitialContext().rebind() ) and by all Tymeac Clients in CosNaming.lookup() (javax.naming.lookup() ). You may change this to your installation requirements.

Alternate:

This field is for non standard Communication. 

private static String tymeac_alternate_class_name -- The name of the non standard communication protocol class.

By default null.

Jini is such a protocol. Since there is no way to package such a method, the execution is up to the user. You must write a class that implements your way of communication.  Place the name in the String tymeac_alternate_class_name. Set tymeac_use_alternate to true.

When both these fields are set as such, the user-maintained class TySvrComm loads the alternate class and invokes the getTymeacInterface() method to retrieve a reference to the remote object. The internal Tymeac Client class that uses the GUI and command line classes also uses this identical procedure.

We provide an example of how to do this in the samples, (TyDemoJiniAlternate).

Instance Methods:

public String getTymeac() -- Accessor, returns String tymeac_name.

public String getUrl() -- Accessor, returns String tymeac_url_name.

public String getIIOPName() -- Accessor, returns String tymeac_iiop_name.

public String getAltName()  -- Accessor, returns String tymeac_alternate_class_name.

public boolean useAlt() -- Accessor, returns boolean tymeac_use_alternate.

public boolean useIIOP() -- Accessor, returns boolean tymeac_use_iiop.

public boolean useRMI() -- Accessor, returns boolean tymeac_use_rmi.
