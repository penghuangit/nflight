Custom Socket Factory Set Up

In this section we discuss the custom socket factory set up procedures. If you used Tymeac CSF prior to Release 4.0 then you must upgrade.

For the non-activatable server:

Tymeac supports your custom socket factories by providing a method createFactory() in the server class: com.tymeac.base.CSFServer.

You must create a class that extends com.tymeac.base.CSFServer and override the createFactory() method. We supply a demonstration for this in class:
    com.tymeac.demo.CSFServer

This demo class extends com.tymeac.base.CSFServer. Has the identical createFactory() since this must be your code.

For the activatable server:

You may create your custom socket factories in the constructor used during activation. We supply a sample in module: com.tymeac.demo.TyDemoActivationCSFactory. This class extends com.tymeac.base.TymeacActivationImpl.

public TyDemoActivationCSFactory (ActivationID id, MarshalledObject data)
   throws  RemoteException,  Throwable {

// the parameters for TymeacActivationImpl
super(id,
            new TyDemoClientSocketFactory(),
            new TyDemoServerSocketFactory(),
            data);

Alternatively, you may specify the custom socket factories in the setup at the export step. Since this is custom code we do not provide an example. You must use:
    java.rmi.activation.Activatable.exportObject(
                            Remote obj,
                            ActivationID id,
                            int port,
                            RMIClientSocketFactory csf,
                            RMIServerSocketFactory ssf)

in place of the Activatable.register() in Class com.tymeac.demo.TyDemoActivationBase, method createBackend().
