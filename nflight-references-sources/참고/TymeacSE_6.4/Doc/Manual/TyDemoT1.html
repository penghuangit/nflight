                 swing-layout1.0.4.jar
The script we supply has this in the classpath.

We supply this jar file in the classes\lib directory. The script file for this class includes the additional jar file in the classpath.

To build this class using Eclipse or another IDE, copy the classes\lib directory to your project. Such as:
\workplace\YourProject. Then add the additional jar file to the project.

Elements:

Function Name:   Enter a Function name from Function_1 to Function_8.  These are described in the Demonstration System.  

Note that these are 10 characters each. The text field converts to a String. Therefore, do not include any white spaces after the Function name as it will not match the Demonstration Functions.

A1-3:    Enter any two digits from 00 to 99.
B1-3:    Enter any two digits from 00 to 99.
C1-3:    Enter any two digits from 00 to 99.  

Result:   This displays the result of a Request to Tymeac Server described  in the Demonstration System.

Buttons:

SEND:   Press to send the request to Tymeac Server.

Class com.tymeac.demo.jframe.TyDemoT1Bean is a complement to T1. 

public void readLn()

takes the Function Name, digits for A, B, and C, calls the goTymeac() method and sets the result text. 

private Object[] goTymeac()

sets up the TymeacParm and calls the methods in TySvrComm for the Tymeac Server Functions.

Tymeac Functions 1-4 and 8 use a Synchronous Request.  That is, the client waits for the completion of the requested Tymeac Function.  The return message from Tymeac is: Tymeac SR(0000) + the result of the processing.

Tymeac Functions 5-7 use an Asynchronous Request.  That is, the client does not wait for the completion of the requested Tymeac Function.  The return message from Tymeac is: Tymeac AR(0000)[MilliTime,RequestId]
