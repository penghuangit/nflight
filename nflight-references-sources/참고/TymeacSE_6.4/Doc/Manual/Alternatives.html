Alternatives for Statistics and Logging

In this section we discuss the alternatives for Tymeac statistics and logging.

Tymeac gathers statistics either on request or at shut down and sends the report to one of three mediums.  Tymeac develops messages as they occur and sends the messages to one of three mediums.

The order Tymeac uses at Tymeac start up is:

  1. If a DBMS Table name exists, use it and ignore the other fields.
  2. If a File name exists, use it and ignore the alternatives fields.
  3. If an Alternatives name exists, use it.
  4. When none of the above, ignore the option, (statistics or log).

DBMS

The statistics table definition is here. The logging table definition is here. Tymeac uses a Configuration File to get the names of these tables at start up and when necessary for use.

Tymeac verifies the existence of the tables at start up.

The DBMS is the first in order for checking for these options. When a table name exists in the Configuration File, then Tymeac ignores the other two options.

Files

The file definition is either within the Configuration File or within the User Variables class.

When using a Configuration File, this is the second option Tymeac checks at start up. Therefore, the DBMS table name must be null.

When using the User Variables class, this is the first option Tymeac checks at start up.

Tymeac verifies the existence of the files at start up by opening each file.

When a file name exists, then Tymeac ignores the last option.

Classes

For those times when neither a DBMS nor a File is suitable for your needs, Tymeac provides a do-it-yourself method. When one of the other options, above, is present, Tymeac ignores this option. The class definition is either within the Configuration File or within the User Variables class.

The class must implement the TymeacAlternativesInterface:

public interface TymeacAlternativesInterface {
public boolean verify();

public boolean write(String msg);

The class must have:

  • a no-argument constructor
  • a verify method
  • a write method

Tymeac loads and instantiates the class at start up. Tymeac then calls the verify method. When the return from the verify method is true, then Tymeac uses the write method as necessary.

We provide several classes that demonstrate this option in Samples.
