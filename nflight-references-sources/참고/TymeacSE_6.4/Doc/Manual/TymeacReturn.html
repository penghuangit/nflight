Tymeac Server Return Values

This version of Tymeac goes back to JDK1.1. This class should have been part of the original release, but better now than never.

Tymeac returns an Object[] from both the syncRequest() and the asyncRequest(). The details are here. Working with those Objects is sometimes tedious and prone to error. Therefore, this class extracts necessary information from those Objects and puts it into this class for easy reference. Changing the server to return this class instead of the Object[] would make users very unhappy.

The class has one public static method, formatCallReturn(Object[]).
    Simply pass the return Object[] from a Tymeac call and receive a formatted class in return.

TymeacReturn value = TymeacReturn.formatCallReturn(back); // where back is the Object[] from the Tymeac call.

The class has five getter methods for data from the call:

public int getReturnCode() The return code as found here. The formatting errors are part of that document detailed here.

The significant formatting error number is 9001 - indicating that the return object from Tymeac is null. This can happen when the communication link breaks in a remote environment.

Numbers greater than 9001 indicate some type of formatting error which is probably system damage on the part of the server or corruption of the returned objects in the network. The exact detail of such may be irrelevant for most applications.

public String getTyMessage()  The String from the first Object in the Object[]. This may be useful for error recovery routines.

public long getSessionId()  For async calls, the session id for use in canceling a call.

public long getRequestId()  For async calls, the unique id for use in canceling a call.

public Object getUserData()  For sync calls, the user data from the user processing.

See for example: com.tymeac.demo.TyDemoInternal
