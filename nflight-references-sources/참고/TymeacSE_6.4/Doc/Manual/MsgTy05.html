Messages: Scheduling Ty05

This section details the Request Brokering messages.

(P) Printed to System.out.println() (this is optional).
(L) Sent to the Tymeac Log.
(N) Sent to the Tymeac Notification Function.

010C (P) (L) (N)

'TymeacInternalCancel' Queue failed scheduling. Reason=xxxx

Tymeac does callbacks in Synchronous Requests by using an internal Tymeac Function (TymeacInternalCancel), internal Queue (TymeacInternalCancel) and internal Processing Application Class (com.tymeac.base.TymeacInternalCancel). All the internal Classes are loaded at Tymeac start up.

The Queue failed to schedule. The reason (xxxx) for the failure can be found in the usage document. There is system damage.

The Processing Application Class executes using try-catch blocks. The caught exceptions are ignored. The calling Client may not be there; the request may finish before the callback can execute. These are normal occurrences and are rightfully ignored. Other network type errors are beyond what we can handle since this request executes in the background.

Errors are not caught. These can cause Tymeac to disable the Queue Thread.

The Server continues.

The calling request is given a general notification that the callback failed.
