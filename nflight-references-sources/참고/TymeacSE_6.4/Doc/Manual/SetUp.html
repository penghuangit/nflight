Tymeac Set Up

If you do not use Jini, then you should delete the Jini source and class files to save aggravation with compiler errors.

com.tymeac.jini
com.tymeac.demo.jini2
com.tymeac.serveruser.JiniShutDownExit

When you edit the UserQueues and UserFunctions you may also remove the "J" queues and functions and then remove the "J" source and class files from the com.tymeac.serveruser directory.

Tymeac may run as a non-activatable remote object or
    an activatable remote object or
    a Jini Service or
    as an embedded, non-RMI Server.

It is necessary to first establish that Tymeac installed correctly. This is the first set up. If you intend to use RMI, please do this first. Even if you cannot wait to try the activatable remote object and the Jini examples, they may fail unless your environment is properly set up.

If you do not have a networking (TCP/IP) setup on your computer then you cannot use the RMI servers. You may only use the internal server.
Or if you just don't care about RMI then:
     Go directly to the internal server setup and run the internal server bypassing the others.
