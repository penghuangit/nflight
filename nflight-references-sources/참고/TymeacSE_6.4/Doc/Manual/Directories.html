File Directories

This section details the Tymeac System use of packages and file directories.

Tymeac uses packages and file directories.

The following are the packages:

  • com.tymeac.base
  • com.tymeac.client
  • com.tymeac.client.jframe
  • com.tymeac.common
  • com.tymeac.serveruser
  • com.tymeac.jini.base
  • com.tymeac.jini.client

Additionally, there are packages for the demonstration system:

  • com.tymeac.demo
  • com.tymeac.demo.jframe
  • com.tymeac.demo.jini.base
  • com.tymeac.demo.jini.client
  • com.tymeac.demo.jini2.base
  • com.tymeac.demo.jini2.client

And never forget testing:

  • com.tymeac.test -- See the testing document for usage. These classes are mainly for those who develop Tymeac code.

The file directories into which the product unzips is "your name"\classes\ plus the above format. We refer to the "your name"\classes as <home>.

Tymeac is a client/server pair. You may run Tymeac on a single machine. This is the way most applications use Tymeac. Tymeac is a service for applications. Therefore, it usually runs within an application environment.

However, you may put the Tymeac Server on a separate machine than the Tymeac Client. Because of this option, we segregate the classes to facilitate separate machines. However, your CLASSPATH, for use with the RMI/JNDI Registry, must point to the <home> directory in any case so that it may find the _stub classes.

As part of the shipment are additional directories

  1. Source -- These are the source java files we provide.
    JFrame  These are the source demonstration system Frame classes that use the Swing interface.
  2. Utility  -- These are the non-production classes for checking Queues and Functions.
  3. Script -- These are source scripts for starting Tymeac Server.
  4. Config -- The configuration files for security and Jini.

The Base Directory:

This directory contains all the Tymeac Server classes. This includes the RMI stub classes. See also the RMI background document.

This is the codebase, (java.rmi.server.codebase property), directory where the server's classes will be made available, when downloading via HTTP. 

The Client Directory:

This directory contains all the Tymeac non GUI classes.

The Client\JFrame Directory:

This directory contains all the Tymeac GUI classes that use the Swing interface.  The subdirectories contain the Help html files and the Image files.

The Common Directory:

This directory contains the user-maintained application classes used on both the Tymeac Server and Clients. You may put this directory (with any naming convention) anywhere you wish.

We include the Tymeac Variables class, TyVariable.  The non-activatable Tymeac Server, the activatable object, setup class (for the activatable Tymeac Server) and Tymeac Clients use this class.

The \ServerUser Directory:

This directory contains all the USER Processing Application Classes. You may put this directory, (with any naming convention), anywhere you wish.

  1. This is the place where all user Processing Application Classes belong. However, for the Java2 platform, you may place these classes in a directory outside the classpath and use a URL to locate the classes. See the class naming document.

  2. This is where the base security class, TyServerRMISecMgr, resides. See also security issues.

  3. We include the User Classes. If you are only using a DBMS for storing Function, Queue and Variables information, then you may delete these classes.

The \Jini\base Directory

This directory contains all the Sever code for Jini2.x.

The \Jini\client Directory

This directory contains all the client code for Jini2.x

The \Security Directory:

This directory contains two files: policy.all and rmid_policy.all. These grants all permissions for the security manager and for using rmid for the activation version. You certainly may alter these to any value necessary for your installation. See also security issues.

The \Source Directory:

This directory contains the source code we supply with the Tymeac Product. As of release 6.0 this is a zip file containing the full product source code.

The \com sub directories conform to the packaging standard.

\ForkJoin is the source file for the Fork-join example.

\NetBeans sub directories contain form files for development with NetBeans.

\OldDempGUI contains the old source files for the frame demo classes without using NetBeans.

\Utility contains utility source files.

The \Utility Directory:

This directory contains the classes for checking Queue names and Function names.

The \Scripts Directory:

These directories contains source code scripts for starting the Tymeac Server as well as running the demonstration system. They are .bat files and some.sh files. (We haven't been able to add the Unix .sh files since release 4.x) You must edit these files for your environment.

The top level directory, \Scripts, contains the base files.

subdirectories  --

    /Jini, contains the Jini files for Jini1.x.

        /META-INF/PREFERRED.LIST -- For Jini dynamic policies.

    /SQL -- SQL statements for the DBMS mode

    /Test -- scripts associated with the diagnose() function, see the testing document.

The Config Directory:

This contains the configuration files for security and Jini.

Packages and directories for the Demonstration System:

The Demo Directory:

These are the Tymeac non-frame Client classes for the Demonstration System.  Once you finish the demonstration series you may move these classes to a test machine.

The Demo\JFrame Directory:

This directory contains all the Demonstration System Tymeac GUI classes that use the Swing interface.

The Demo\Jini\Base Directory: (Jini1.x)

This directory contains all the Demonstration System Tymeac Jini Server classes.

The Demo\Jini\Client Directory: (Jini1.x)

This directory contains all the Demonstration System Tymeac Jini Client classes.

The Demo\Jini2\Base Directory: (Jini2.x)

This directory contains all the Demonstration System Jini Document Service for Jini2.x Server classes.

The Demo\Jini2\Client Directory: (Jini2.x)

This directory contains all the Demonstration System  Jini  Document Service for Jini2.x Client classes.
