Messages: Start Up Ty00

This section details the Start Up messages.

(P) Printed to System.out.println(), (this is optional).
(L) Sent to the Tymeac Log.
(N) Sent to the Tymeac Notification Function.

001 I (P) (L)

Tymeac Server Initializing in [DBMS][Stand Alone] mode.

The server is starting either in DBMS or Stand Alone mode.

003 I (P) (L)

Tymeac Server, no Configuration File found for start up.

DBMS mode is the default for the Server.

The Server requires a Configuration File (or the command line option -s for stand-alone mode.) The command line parameters specify the Directory [Filename] for the location of this file.

DBMS execution cannot proceed. Tymeac loads the Stand-Alone version (User Classes.)

004 I (P) (L)

The Message Log  for this session is TTT.

Purely informational. TTT is the DBMS Log Table, local file name, or Not in use message.

005 C (P)

Logging is disabled for this session.

A Log was specified in the Configuration File / TymeacUserVariable and there was a DBMS / file error. Errors are printed previously.

006 I (P) (L)

The Statistics repository for this session is SSS.

Purely informational.  SSS is the DBMS Statistics Table, local file name, Alternate Class name or Not in use message.

007 C (P) (L)

Statistics are disabled for this session.

A Statistics repository was specified in the Configuration File / TymeacUserVariables and there was a DBMS / file error. Errors are printed previously.

008 S (P) (L)

Class Loader Security Exception

Tymeac uses a class loader for all Tymeac management classes. You must allow this in the security policy file.

Execution stops.

009 S (P) (L)

Tymeac Server, RMI [JNDI] rebinding failure

The RMI [JNDI] Naming.rebind() failed. Errors are printed. An unMarshaling error for the class: Tymeac[Standard][IIOP]Impl_Stub indicates the RMI [JNDI] Registry cannot find this class. You must set a CLASSPATH or CodeBase for this class. See the RMI background document.

Execution stops

011 I (P) (L)

The Notification Function for this session is NNN.

Purely informational. NNN is the Notification Function name.

012 C (P) (L)

Notification is disabled this session.

There was an RMI error, or, the Tymeac Function could not compete. Errors are printed for RMI, previously.

014 C (P) (L)

The Monitor is not in use.

The Monitor interval was specified as zero. Since the monitor is an integral part of Tymeac, not using it requires a conditional notice.

044 S (P) (L)

Tymeac Server Function index position= [value of index starting at 0] has no value.  Startup cannot continue.

The Function position is null.  This usually is the result of Class - Tymeac[User][Internal]Functions constructor, functions array, not having a value entered into it.   For example, when adding a new Function, it is easy to copy and paste one that already exists and then simply make changes.  However, the index value must be incremented along with the other changes.  When this later step is left out, two Functions occupy the same array position and the subsequent array position is null: 
functions[0] = new TymeacIndividualFunction("old values ...)
functions[0] = new TymeacIndividualFunction("new values ...)

Execution stops.

046 I (P) (L)

Tymeac Server, at least one Function generated a duplicate hash code. Using a String compare algorithm.

Tymeac tries to use  a hash code algorithm for Function name look up. If two Function names generate the same hash code then Tymeac is forced to use a String compare algorithm. Strings are slow compared to hash codes. See the Function naming document for further insight and solutions.

Execution continues.

070 I (P), (L)

Tymeac Server RMI rebinding variables: tymeac_name [on port=xxxx]

Tymeac Server is attempting an RMI Registry rebind with the field from the TyVariables Class: tymeac_name The main reason this name prints here is for the log file. It's a way of identifying the particular start up. This becomes important when the same file/table is used for many start ups.

When a -port command line option was specified, then that port number prints here.
