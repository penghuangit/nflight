Tymeac User Class

This section details the User Classes. These Classes are for saving the Tymeac Variables, Queues and Functions when no DBMS is available.

This is also known as the Stand-Alone Version.

Tymeac can run on any processor that supports Java. However, not all environments support a relational data base management system or access to a local file. This is why we supply the User Classes.

In order to use Tymeac without a local file or DBMS, we supply three (3) user maintained Classes. All these Classes reside in the <home>/com/tymeac/serveruser directory (or whatever you wish to call it.)

com.tymeac.serveruser.TymeacUserVariables

com.tymeac.serveruser.TymeacUserQueues

com.tymeac.serveruser.TymeacUserFunctions

These Classes are the replacement for the tables in a DBMS and the configuration file.

We ship these Classes filled in for use as a Demonstration System.

These User Classes are a manual method and somewhat prone to error. Sometimes it's easy to manually alter the java file and recompile rather then use the GUI.
