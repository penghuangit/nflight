Is This The Right Product For You?

 

This is the standard edition (Java SE version 1.6+) component oriented Fork/Join product.

  • Tymeac separates your request into it's individual components.
  • Tymeac places each component onto a Queue, serviced by a thread pool.
  • A thread in each thread pool executes the component and returns a result to the server.
  • Tymeac joins the individual components together and returns it to the caller.
  • Tymeac supports both synchronous and asynchronous requests.
  • Additionally, Tymeac also supports agents for asynchronous requests.
  • Using Tymeac is like programming completely separate components all running simultaneously.

If you need a standard edition that runs on the Java Micro Edition, then download TymeacME.

Also available for download is the Divide-and Conquer product (TymeacDSE); also known as scatter-gather.

  • TymeacDSE allows you to break up your request into small, identical tasks according to your thresholds
  • TymeacDSE scatters the tasks to every thread in the system
  • Your code implements two abstract methods in class, Task: compute() and complete()
  • Your code executing inside the compute() method either creates more tasks or returns an object
  • When all the tasks finish processing, TymeacDSE calls the complete() method passing the list of objects from the compute() methods. 

Divide-and-Conquer is best known in the compute intensive, scientific/academic arena. That is, when each task does a small, compute intensive function such as summing an array of numbers.

Divide-and-Conquer does very well with sorting and map/reduce.

Phase one is to split the large array into many smaller arrays for sorting or mapping. When the split array is small enough, it is processed in the sub-task.

Phase two is to merge or reduce the sorted lists.

Divide-and-Conquer is the sweetheart of massive data processing on massive multi-core processors where each task does identical work. If you can run this product on 16, 32, 64 CPU machines then you are going to love TymeacDSE.

Divide-and-Conquer doesn't do very well on small numbers of processors or small amounts of data. On a dual-core processor, it takes more time to break up the request then it does to simply process the array sequentially. If you only have a small array, why bother with additional work?

Divide-and-Conquer doesn't do very well when each element in an array needs different processing or needs to access outside resources. There are no restrictions on accessing outside resources or inter-task communication. It's just the waiting and resource coordination can take a lot of time

If you need a standard edition that runs on the Android platform (API Level 8 and above), then download TymeacAND.
