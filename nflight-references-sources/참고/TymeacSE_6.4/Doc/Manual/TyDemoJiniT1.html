This source code supplied Class demonstrates Tymeac back-end functionality.

Tymeac is not a Jini Service nor a Jini Client. Tymeac is an activatable remote object. This series of Classes demonstrates how Tymeac may be used as a back-end process in a Jini network.

The Swing Class was generated with Symantec's VisualCafe.

  (click to link to full image)

Elements:

Document Name:   Enter a document name.  No document need exist.  Nothing is read.

Result:   This displays the result of a Request to the TyDemoJiniDocService.  This Service uses Tymeac Server as a back end.  The Tymeac Server function is JFunction_1.  This uses four Queues, JQue1-4.

Buttons:

Send & Wait:   Press to send the request to the TyDemoJiniDocService and wait for completion.

Send No Wait:   Press to send the request to the TyDemoJiniDocService and not wait for completion.

Class com.tymeac.demo.jini.client.JTest1Bean is a complement to JTest1. 

public void sendNoWait()

takes the Document Name, calls the goHost() method and sets the result text.

public void sendWait()

takes the Document Name, calls the goHost() method and sets the result text.

private String goHost()

gets a new instance of TyDemoJiniDocAlternate(), gets the backend remote object from the method getTyDemoJiniDocInterface(), goes to the backend service with:  backend.putDoc(document, waiting)

Returns the result.
