Tymeac Messages

This section details the Tymeac Message structure.

STRUCTURE

Messages have a message number followed by text.

All message numbers begin with the letters, Ty.

The next two digits identify the component:

00 - Start up
01 - Start up, extended for the Configuration Provider
02 - Shut down
03 - Monitor
04 - Request Brokering
05 - Scheduling
06 - Queue Thread
07 - Output Agent Thread
08 - General Tymeac Component
09 - User Modification

The next three digits is a unique number within each component.

The last letter denotes the severity of the message:

I - Information
C - Conditional
S - Severe

FORMAT

Messages are grouped by component.  Messages are sent to any of three mediums: (P) Printed to System.out.println(), (this is optional).  (L) Sent to the Tymeac Log. (N) Sent to the Tymeac Notification Function.

The message unique identifier and severity are followed by a (P), (L), (N) to denote where the message is sent.
This is followed by the Text of the message.
Any explanation and recommendation follows thereafter.

E.G.
Ty00001I (P) (L)
Tymeac Server ..................

is a Tymeac message for the Start up.  The message number is 001.  It is an information only message.  It is printed and sent to the Log.
