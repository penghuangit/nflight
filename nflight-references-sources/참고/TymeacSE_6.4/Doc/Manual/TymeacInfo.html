Tymeac Run Time Information

This is a Singleton Class containing various information on the currently executing Tymeac Server

Tymeac Server instantiates this class during start up. The instance of this class is available to all user classes by the getInstance() method.

JavaDoc:  public final class com.tymeac.base.TymeacInfo

Class Methods:

public static com.tymeac.base.TymeacInfo getInstance()

This static method returns the instance of the TymeacInfo class.You may then use the instance methods to retrieve information about the currently executing Tymeac Server.

Instance Methods:

public String getDBUrl()

Returns a String of the URL name when using DBMS mode, or null.

public String getDBDriver()

Returns a String of the Driver Manager name when using DBMS mode, or null.

public String getDBUserid()

Returns a String of the User Id when using DBMS mode, or null.

public String getDBPassword()

Returns a String of the Password when using DBMS mode, or null.

public String getDBShutdown()

Returns a String of the shut down class name when using DBMS mode and an argument was specified at startup, or null.

public long getMilliTime()

Returns a long that is the Tymeac startup time.

MilliTime is the System time in milliseconds since 1970. This time stamp appears on the start up message, on the shut down message, as an instance field in Class CancelParm (see how to cancel synchronous requests -- syncReq() in the cancel document) and within all return messages for Asynchronous Requests. It is the way to identify this session of Tymeac.

public TymeacInterface getTymeac()

Returns a reference to the Tymeac Server after Tymeac Start Up finishes.

When startup creates this class, the Server is not fully available. That is, exporting the remote object for the external version or saving a reference to the Server implementation for the internal version has not completed yet. Naturally, the base must be built before it can be saved/exported. Therefore, when used in a startup exit, the return value may be null. Simply, retry the method at a later time.

public String getVersion()

Returns a String that is the current version of the Tymeac Server. This is in the form: 1.1.3 or 2.1 etc.

public Object getJoinManager()

Returns an Object (JoinManager reference) for use by Jini modules. You must cast this to a net.jini.lookup.JoinManager. It is saved as an object since not all developers use Jini and may not have the Jini packages.

public int getStatus()

Returns an int that is the current status of the Tymeac Server.

public final static int INITIALIZATION = 0; // 0 in init
public final static int AVAILABLE = 1;         // 1 normal processing
public final static int SHUT_STAGE1 = 2;    // 2 shutdown stage 1
public final static int SHUT_STAGE2 = 3;    // 3 shutdown stage 2
public final static int SHUT_FORCE = 4;     // 4 shutdown force
public final static int SHUT_COMPLETE = 5;   // 5 shutdown complete

N.B. You should not depend on shut down stages going from 1 to 2 to 3 etc. During stage 1, if there are no Queue Threads active then the shut down continues past the Stage 1 user exits into the Stage 2 user exits, the embedded database user exit and finally termination. There is no reason to change this variable.

public int getStartType()

Returns an int that is the start up type for this Tymeac Server. (CSF is Custom Socket Factory)

public final static int NOT_STARTED = 0;    // 0 initializing
public final static int NON_ACT_RMI = 1;    // 1 non-activatable RMI
public final static int ACT_RMI = 2;            // 2 Activation RMI
public final static int IIOP = 3;                     // 3 IIOP start up
public final static int INTERNAL = 4;          // 4 Internal Server
public final static int NON_ACT_CSF = 5;   // 5 non-activ CSF RMI
public final static int ACT_CSF = 6;             // 6 Activatable CSF RMI
public final static int NON_ACT_JINI = 7;    // 7 non_activatable Jini
public final static int ACT_JINI = 8;          // 8 Activatable Jini
public final static int ACT_CSF_INI = 9;   // 9 Activatable CSF Jini
public final static int IIOP_POA = 10;     // 10 Portable Object Adapter

public int getShutOrigin()

Returns an int that is the origin of the shut down.

public final static int DEACTIVATION = 1;            // 1 from deactivation
public final static int SHUT_REQUEST = 2;            // 2 from request

When using the activation framework, Tymeac Server deactivates when the idle time reaches the threshold. Before deactivating, the Tymeac Server calls the shutdown classes, when present.

Examples:

If one wishes to use several methods of the TymeacInfo Class within a Processing Application Class then one may save the instance of the TymeacInfo Class and use the instance methods when necessary.

TymeacInfo ti  = TymeacInfo.getInstance();

System.out.println("Tymeac MilliTime=" + ti.getMilliTime()):

System.out.println("Tymeac Version is  " + ti.getVersion()):

If one wishes to use a single method of the TymeacInfo Class within a Processing Application Class then one may get the instance of the TymeacInfo Class and use the method immediately.

if  (TymeacInfo.getInstance().getShutOrigin() == TymeacInfo.DEACTIVATION) {

// Shutting down because of deactivation
