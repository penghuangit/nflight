SET_ENV -- These are the environment variables. You must alter these for your location.

TYMEAC_HOME     This is the place you installed Tymeac plus the Classes directory. You must have the /classes (or \classes) at the end. We do not ship jar files. All the classes are in the Classes directory.
usr/local/tymeac4.0.1/classes
x::\tymeac4.0.1\classes

default: none

 

TYMEAC_IIOP_PORT     This is for use with the IIOP/POA server. This port number must be in sync with the orb daemon or transient name server (tnameserv for backward compatibility)

orbd -ORBInitialPort nameserverport options [ commands ]
tnameserv  (the default port.is 900)
---------------------------------------------------------------------------------

The Well Known Ports:
omginitialrefs 900/tcp OMG Initial Refs
omginitialrefs 900/udp OMG Initial Refs

The default is 900.

 

TYMEAC_RMI_PORT      This is the optional exporting port number for the RMI/CSF Servers.When no port number is specified, Tymeac exports to the anonymous port.
----------------------------------------------------------------------------------
The port numbers are divided into three ranges: the Well Known Ports,
the Registered Ports, and the Dynamic and/or Private Ports.

The Well Known Ports are those from 0 through 1023.
The Registered Ports are those from 1024 through 49151
The Dynamic and/or Private Ports are those from 49152 through 65535 -- This is where you should pick port numbers for Tymeac Server.

The default is 50001.

 

TYMEAC_HTTP_PORT     Codebase http port
This is the port number for the codebase http server This is the same port number you use to start your http server. For Jini this may be:

  -jar %JINI_HOME%/lib/tools.jar -port 8080

The default is 8080.

 

SET_JINI -- These are the environment variables for using Jini. You must alter these for your location.

JINI_HOME     This is the place where you installed Jini

These are appended to the classpath TYMEAC uses for running.

default: none

 

SET_DERBY -- These are the environment variables for using Derby database. You must alter these for your location.

DERBY_PATH     These are the paths (one for each) to each of the Derby jar's:

lib/derby.jar
lib/derbynet.jar
lib/derbyclient.jar
lib/derbytools.jar

These are appended to the classpath TYMEAC uses for running.

default: none

 

SET_HSQL -- These are the environment variables for using Hsql database. You must alter these for your location.

HSQL_PATH     This is the path to the HsqlDB jar:

lib/hsqldb.jar

These are appended to the classpath TYMEAC uses for running.

default: none

 

RMI Server -- The Tymeac Server using RMI

This launches the Tymeac Server using the RMI Registry.
    com.tymeac.base.RMIServer -port %TYMEAC_RMI_PORT% -s

Prerequisites: The RMI Registry must be running and have access to the tymeac stub (com.tymeac.TymeacInterface).

Command line options:
-port 'port number' (export the remote object on this port)
-s (run in stand alone mode, no DBMS)

Derby Server -- Same as above, but with the DERBY_PATH appended to the classpath.

Hsql Server -- Same as above, but with the HSQL_PATH appended to the classpath.

Jini Shut -- A Client shut down using Jini

This launches a Client that accesses for shut down of the Tymeac Server by way of a Jini lookup
    com.tymeac.jini.client.JiniClientShutDown

Prerequisites:
    An http server must be running.
    A look up service (reggie) must be running.
    The RMI Registry must be running.
    [The activation system must be running for activation.]
    A Tymeac Server Service must be running

Command line options: (3)
    -config
    config\tymeac-jrmp-client.config
    -secure

Security: When security is active, the -secure option, above, brings up a dialog window asking for the pass word. You must enter -- tyclient (in lower case.)

CfgToXml -- Conversion for the old cfg file to new xml format. The class is in the default directory (no package.)

This launches a standalone program that converts the file format. It uses a dialog to get the directory and filename of the old and new files.

Prerequisites:
    none

Command line options:
    none
