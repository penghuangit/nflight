Login Servers

The RMI Server builds it's regular remote object and exports it to the run time. The RMI Server also builds a login remote object that implements a simple interface with one method: login().

The RMI Server binds that remote object to the RMI Registry. When a client retrieves the remote object using the standard name, the client gets the login remote object. The client sends login information to the login method of the Server.

If the Server approves, the server then returns the regular remote object to the client. The client then communicates normally with the Server.

The source code is not provided separately in the compiled download. Since the LoginServer extends the RMIServer, having one without the other is unworkable. The source code may be downloaded from the place where you downloaded the product.

We implement this scenario for you with both the RMIServer and the CSFServer. We do not provide this scenario for the IIOP servers. Using ORBs may require a much different procedure.

public interface TymeacLogin

public TymeacInterface login(Object[] obj)
       throws java.rmi.RemoteException;

The implementation, for demonstration purposes only, simply returns the Tymeac remote object:

// save pointer

    } // end-constructor

     public TymeacInterface login (Object[] obj )
        throws RemoteException {

// Say we got here
msgOut("Did a user login", 10);

// add logic before
return T.getTi();

    } // end-method

The Server extends com.tymeac.base.RMIServer[or CSFServer] and overrides the rebind() method. to export the login remote object. We provide working source code to accomplish this scenario. We also provide the stub class (and source) so you can run right away.

The names of the servers are:
        LoginCSFServer
        LoginRMIServer

The names of the clients are:
        DemoClientBaseLogin
        DemoClientShutLogin

The names of the client scripts are:
        LoginCSFServer.bat -- starts the Custom Socket Factory RMI Server
        LoginRMIServer.bat -- starts the Standard RMI Server
        LoginShut.bat -- does a shut down
