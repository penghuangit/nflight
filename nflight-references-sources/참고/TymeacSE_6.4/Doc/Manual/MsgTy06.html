Messages: Queue Thread Ty06

This section details the Queue Thread messages.

(P) Printed to System.out.println(), (this is optional).
(L) Sent to the Tymeac Log.
(N) Sent to the Tymeac Notification Function.

005 S (N) (L) (P)

Thread [name] in Queue [name] disabled by IllegalAccessException

The thread uses a try block on: java.lang.reflect.Method.invoke() to access the Processing Application Class.

See also return code.

This exception was caught.

010 S (N) (L) (P) 

Thread [name] in Queue [name] disabled by InvocationTargetException

The thread uses a try block on: java.lang.reflect.Method.invoke() to access the Processing Application Class.

See also return code.

This exception was caught.

015 S (N) (L) (P)

Thread [name] in Queue [name] disabled by IllegalArgumentException

The thread uses a try block on: java.lang.reflect.Method.invoke() to access the Processing Application Class.

See also return code.

This exception was caught.

020 S (N) (L) (P)

Thread [name] in Queue [name] disabled by ThrownException

The thread uses a try block on: java.lang.reflect.Method.invoke() to access the Processing Application Class.

See also return code.

This exception was caught.

025 C (L) (P)

Thread nnn in Queue qqq Output Agent Scheduling failure=xxxx, Request rrrr is stalled.

All Queues in the List of Queues for the Function finished processing. While attempting to schedule the Output Agent Queue for Request Id rrrr, the schedule failed. xxxx is the return code.

nnn is the Queue Thread number.

qqq is the Queue name.

026 I (L) (P)

Request rrrr, Output Agent is [name] 

This is a compliment to 025.  Match the rrrr here with the yyyy in 025.

The reason for the second message is that the 6024/5 message is very long.

090 S (L) (P)

Thread [name] in Queue [name] Expunged

The Tymeac status of the thread was "disabled" (see how threads become disabled.)

An administrator used the ENABLE ALL button on the Queue Thread display to enable all disabled threads. This button sets the instance reference of the thread (in the thread management structure) to null. A new thread may or may not have taken the place of the thread writing this message.

When the thread writing this message resumes, it finds its instance reference either null or that another thread has taken its place, it writes this message and terminates quickly (exits in the run() method.)

901 S (L) (P) (N)

Thread [name] Failed with uncaught Exception=xxxx

This exception (xxxx) was caught by the uncaught exception handler.

The thread is logically disabled and the physical thread terminates.
